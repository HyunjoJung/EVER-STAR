# 시스템 아키텍처 정리

![시스템 아키텍처](images/image_0048.png)

## 변경 메모
- 정현조가 아키텍처를 다시 정비하면서 **git에 노출된 개인정보를 제거하고 외부 의존성(배포 대상 서비스 버전/키) 변화**에 맞춰 재구성을 시도했습니다. 이를 위해 저장소를 초기화하고(원격 제거, `git init`), 민감 정보가 섞인 인프라/설정 파일은 정리·삭제했습니다.
- 그러나 외부 서비스 키 재발급·버전 확인, 배포 파이프라인 재구성이 필요해 작업이 중단된 상태입니다. 재시도 시에는 ① 비공개 설정/키를 분리한 샘플 env 템플릿 정리, ② 의존성 버전(SaaS·라이브러리) 재확인, ③ 배포·테스트 파이프라인을 새로 잡는 순으로 진행해야 합니다.
- 모니터링 스택(프로메테우스/그라파나) 관련 파일·의존성은 현재 저장소에서 제거된 상태입니다(코드/compose/의존성 검색 결과 없음). 필요 시 별도 compose·계정으로 재구성해야 합니다.

## 구성 개요
- **Frontend**: React/TypeScript, PWA(알림·이미지 첨부), Storybook/아토믹 디자인, MSW(Mock)로 로컬 테스트, OpenVidu(화상 통화).
- **Backend**: Spring Boot, JPA, MySQL. SSE/스케줄러로 알림 처리, Kafka 언급. OAuth(Kakao/Google) 지원.
- **AI/외부 서비스**: GPT API(편지·퀘스트 답변), 네이버 감정분석 API(감정 → 날씨 표현).
- **알림**: Firebase Cloud Messaging(실서비스용). 로컬/E2E는 비활성화하여 권한 오버레이를 막음.
- **기타**: 파일 업로드/이미지 처리 등은 프론트(PWA)와 백엔드 연계로 처리.

## 운영 시 유의
- 배포용 알림·API 키는 별도 설정 필요. 로컬/테스트에서는 FCM 비활성화 상태.
